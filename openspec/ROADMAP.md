# Thinkrium 開發計劃評估

## 1. 專案願景

**Thinkrium 思藏** = 類似 Obsidian 架構的個人知識庫應用

> **核心原則**：所有內容以**本地 Markdown 文件**儲存，SQLite 僅作為索引層。

### 數據架構
```
RSS 來源 → Feed 模塊 → 保存為 MD 文件 → Library 索引 → 專案管理
                           ↑
                    真實數據源：本地 MD
                    索引層：SQLite（FTS + metadata）
```

### 約束檢視
| 約束                | 狀態           |
| ------------------- | -------------- |
| 離線使用            | ✅              |
| **MD 為真實數據源** | ⚠️ 流程未完整   |
| SQLite 為索引層     | ✅ 架構設計完成 |

## 4. 調整建議

原計劃過度聚焦 UI 細節，建議調整優先級：

```mermaid
graph LR
    A[UI 原型] --> B[MD 保存流程]
    B --> C[MD → SQLite 同步]
    C --> D[Library 真實數據]
    D --> E[端到端驗證]
```

### 推薦下一步

**完善「MD 為核心」的數據流程**：

1. Feed 項目 → 保存為 MD 文件（含 frontmatter）
2. Library 掃描 MD → 建立/更新 SQLite 索引
3. Library 顯示真實筆記
